以下为你整理了最匹配 Vue 3 规则/查询构建器的 GitHub 项目地址：
## 一、 规则树构建器 (Vue 3 专用库)

* 
* arsenaltech/vue3-query-builder
* 地址：https://github.com/arsenaltech/vue3-query-builder
   * 特点：纯正的 Vue 3 规则构建器。完美支持多层级嵌套、拖拽重新排序，并且内置 TypeScript 支持，是最适合直接拿来做条件规则配置的项目之一。 [1] 
* orakley/vue3-query-builder
* 地址：https://github.com/orakley/vue3-query-builder
   * 特点：基于 Vue 3 开发的高级查询构建组件，专门解决“与或非”的多维条件组合。 [2] 
* dabernathy89/vue-query-builder
* 地址：[github.com](https://github.com/dabernathy89/vue-query-builder)
   * 在线 Demo：https://dabernathy89.github.io/vue-query-builder/
   * 特点：这是该领域最经典的 UI 设计框架。虽然底层是老版本，但如果你想仿照它的界面逻辑并在 Vue 3 中自己实现，可以直接抄它的层级 DOM 和 JSON 输出设计。 [3, 4]

## 二、 图形化拖拽编排引擎 (更复杂的 "If-Then-Else" 流程)
如果你的“如果...那么...否则”逻辑不仅仅是表单层级，还要有复杂的流程分支走向，建议使用大厂开源的专业连线画布：

* 
* bcakmakoglu/vue-flow
* 地址：[github.com](https://github.com/bcakmakoglu/vue-flow)
   * 特点：Vue 3 生态里最强大的节点连线库。可以通过自定义节点轻松做出分支节点（If），根据不同条件拉出多条分支（Then/Else）。
* didi/LogicFlow (滴滴开源)
* 地址：[github.com](https://github.com/didi/LogicFlow)
   * 特点：专注于业务审批、规则编排的流程图引擎。非常契合国内的流程场景，提供了成熟的 Vue 3 适配方案。

## 三、 纯逻辑核心解析（无 UI）

* 
* CacheControl/json-rules-engine
* 地址：[github.com](https://github.com/CacheControl/json-rules-engine)
   * 特点：你可以直接把上面任何一个 Vue 3 UI 库生成的 JSON 传递给这个引擎，它负责在 JS/Node 环境里执行具体的规则判断。 [3] 
* microsoft/RulesEngine (.NET 规则引擎)
* 地址：https://github.com/microsoft/RulesEngine
   * 特点：如果你的后端是 C# / .NET，这个是微软官方推荐的极其强大的逻辑规则引擎，完美对接前端的嵌套 JSON 结构。 [5]
