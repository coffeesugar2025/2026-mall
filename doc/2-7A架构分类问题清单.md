# 从0到1识别系统
7A架构：业务架构、应用架构、数据架构、开发架构、运行架构、技术架构、安全架构

## 一、业务架构
1. 系统所在行业的情况是怎样？
2. 系统的目标用户是谁？是给高层决策、运营客服还是普通互联网用户使用？
3. 日常平均使用人数多少？高峰期有多少人同时使用？
4. 系统具备什么业务价值？通过哪些指标衡量业务价值？
5. 系统包含哪些功能模块？
6. 系统存在哪些专属领域概念？如何梳理领域模型？
7. 系统核心业务流程有哪些？完整流程怎么走？
8. 系统有哪些非功能性需求（性能、质量、扩展性、安全性等）？
9. 系统未来的迭代发展规划是什么？
10. 运营、客服高频反馈的业务类问题有哪些？

1. 系统各角色对应的操作权限、数据隔离范围是什么？
2. 单据完整状态流转规则是什么？取消、驳回、超时等异常业务流程如何处理？
3. 系统业务边界是什么？哪些业务能力本系统承接，哪些交由上下游系统？
4. 行业术语、系统自定义字段、状态枚举值含义是什么？
5. 是否存在多级审批流程，审批触发条件与驳回重走规则是什么？

## 二、应用架构
1. 系统拆分出哪些子系统/功能模块？模块之间依赖关系如何？
2. 系统对外上下游接口有哪些？各接口对接方、对接负责人是谁？
3. 核心业务流程在代码层面如何实现？如何通过类图、序列图还原流程？

1. 模块间哪些场景同步调用、哪些场景异步消息处理？
2. 内部服务接口、第三方接口入参、出参、异常返回规则是什么？
3. 业务规则存储在哪里（硬编码/配置中心/规则引擎）？
4. 系统是否存在循环依赖模块，带来了哪些隐患？

## 三、数据架构
1. 业务数据持久化存储介质是什么？使用MySQL/Oracle哪类数据库？
2. 系统实体关系如何？能否梳理完整E-R图？
3. 整体业务数据量级多少？是否做分库、分表、分区处理？
4. 系统使用了哪些NoSQL数据库，各自用途是什么？
5. 存在哪些实时/定时数据同步任务，同步链路是什么？
6. 是否接入大数据计算框架，用于哪些统计分析场景？

1. 核心数据表主键、索引、关键字段、枚举状态码含义是什么？
2. Redis等缓存的更新失效策略，如何解决缓存穿透、击穿、雪崩？
3. 冷热数据如何归档、清理？数据备份与恢复流程是什么？
4. 报表、看板的数据来源是什么，数据延迟多久？

## 四、开发架构
1. 系统代码仓库地址在哪里？
2. 工程包如何划分？采用什么分层架构（MVC/Controller-Service-Dao等）？
3. 项目使用了哪些开发框架（SSH、Dubbo等）？
4. 项目引入了哪些第三方工具包（apache commons、guava等）？
5. 项目依赖哪些中间件（消息队列、缓存、定时任务、配置中心等）？
6. 项目依赖公司哪些内部基础平台（权限平台、流程引擎等）？

1. 开发/测试/预发/生产多环境配置如何隔离？配置中心如何使用？
2. 代码分支管理规范、提交规范是什么？
3. 核心模块是否有单元测试、自动化接口用例？本地如何启动调试？
4. CI/CD发布流水线完整流程是什么？

## 五、运行架构
1. 系统常规QPS是多少？峰值可承载QPS上限多少？
2. 系统与上下游交互采用什么方式（RPC/HTTP/消息队列）？同步还是异步？
3. 什么时间段是系统故障高发期（如电商大促）？
4. 关键功能配置了哪些监控项、告警规则？
5. 系统故障如何排查？日志存储位置在哪？是否支持全链路追踪？
6. 系统有无应急管控能力？开关配置、降级、限流如何操作？
7. 系统存在哪些历史遗留坑点？过往线上问题根因是什么？

1. 线程池、异步任务池配置参数是什么？
2. 消息队列生产者、消费者、死信队列处理逻辑是什么？如何避免消息重复消费？
3. 定时任务执行周期、业务作用，如何规避任务重复执行、超时阻塞？
4. 流量突增时的自动兜底策略是什么？

## 六、技术架构
1. 系统划分了哪些部署环境？
2. 生产环境部署多少台机器？
3. 整体部署方案是什么？接入层、集群、分布式架构如何设计？
4. 系统是否容器化部署（Docker/K8s）？
5. 是否采用多机房、异地部署？

1. 网络架构：负载均衡、内外网隔离规则是什么？
2. 发布支持灰度放量吗？故障回滚操作步骤是什么？
3. 服务器CPU、内存、磁盘水位阈值，扩容缩容方案是什么？
4. 存储、网络是否存在单点故障？容灾方案是什么？

## 七、安全架构
1. 系统非功能性需求包含哪些安全约束？
2. 权限平台如何管控账号访问，实现操作安全隔离？
3. 降级、限流、开关等应急操作是否有操作留痕、权限校验？

1. 用户登录鉴权、会话管理方案是什么？
2. 接口是否做签名、防重放、防SQL注入、XSS防护？
3. 手机号、身份证等敏感数据是否加密、脱敏存储展示？
4. 生产高危操作是否需要审批流程？操作日志留存时长？
5. 第三方对接密钥、凭证如何保管轮换？

# 实战落地类问题
1. 熟悉业务和技术后，如何通过迭代需求、修复Bug、代码重构加深系统理解？
2. 学习系统时需要带着哪些思考：为什么这样设计？为什么选用该中间件？是否存在更优实现方式？哪些模块可以优化重构？

1. 接手系统前需要收集哪些交接文档、账号、脚本资源？
2. 如何通过操作页面，反向核对数据库、日志、缓存数据变化，打通全链路？
3. 如何梳理线上故障台账，规避历史重复问题？
