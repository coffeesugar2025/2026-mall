# 网关鉴权机制

> 本文作为速查手册。完整原理与设计讨论见 [专题 02:网关鉴权实战](../07-亮点技术专题/02-网关鉴权实战.md)。

## 总览

| 流量 | Token 形态 | 网关动作 | 撤销能力 |
|------|-----------|---------|---------|
| `/customer/**` | Access Token(JWT,15min)| `JWTUtil.parseJWT` 纯内存校验 | 由 `rs-user` 通过 RT 兜底 |
| `/admin/**` | uuid(2h,滑动续期) | 查 Redis 取真正 JWT,再做权限校验 | `DEL admin:login:token:{uuid}` 即时下线 |

用户端额外配有 **Refresh Token(UUID,7d)**,以 `HttpOnly + SameSite=Lax` Cookie 下发,只在 `/customer/auth/refresh`、`/customer/auth/logout` 使用——网关对 RT 不感知。

## 认证流程

### 用户端(`/customer/**`)

```
1. 客户端请求携带 Authorization: Bearer {accessToken}
2. SecurityFilter 拦截,白名单直接放行
3. 解析 Bearer Token → JWTUtil.parseJWT
4. 校验 claims.tokenType == "access"
5. 提取 userId,写入 exchange.attributes.USER_INFO
6. 路由到下游服务
```

AT 过期后,前端调 `/customer/auth/refresh`(携带 RT Cookie)由 `rs-user` 换新 AT;**网关不参与刷新逻辑**,只认 AT。

### 管理端(`/admin/**`)

```
1. 客户端请求携带 Authorization: Bearer {uuid}
2. SecurityFilter 读取 admin:login:token:{uuid} 取得真正的 JWT
3. JWTUtil.parseJWT 得到 Admin 对象
4. TTL 不足阈值时自动续期
5. checkPermission:超级管理员(role=104)直接放行;其它角色查 AUTH_ROLE:{roleId}:apis Set 做匹配
6. 路由到下游服务
```

## 白名单

以下路径无需认证(常用示例):

- `/customer/auth/login/username`、`/customer/auth/login/phone`
- `/customer/auth/register`、`/customer/auth/refresh`、`/customer/auth/logout`
- `/customer/auth/captcha`、`/customer/auth/captcha/phone`
- `/admin/auth/login`
- `/doc.html`、`/swagger-resources/**`、`/v3/api-docs/**`、`/webjars/**`

白名单由 `ExcludeProperties` 在 Nacos 配置中维护,支持 AntPath 通配。

## Redis Key 一览

| Key | 类型 | TTL | 用途 |
|-----|------|-----|------|
| `user:refresh:token:{uuid}` | String | 7d(滑动) | 用户端 Refresh Token → userId |
| `user:login:time:{userId}` | String | — | 最近登录时间(审计用) |
| `admin:login:token:{uuid}` | String | 2h(滑动) | 管理员 Session,值是真正的 JWT |
| `AUTH_ROLE:{roleId}:apis` | Set | 跟随角色 | 角色可访问的 `METHOD-path` 列表 |

## Access Token(JWT)结构

```json
{
  "jti": "uuid",
  "sub": "{userId}",
  "iss": "sg",
  "iat": 1703116256,
  "exp": 1703117156,
  "userId": 1001,
  "tokenType": "access"
}
```

- HMAC-SHA256 签名,密钥由 Nacos 下发后通过 `JWTUtil.initialize(jwtKey)` 注入
- `tokenType` 用于类型隔离(参见 `JWTUtil.isAccessToken`)

## Refresh Token 结构

RT **不是 JWT**,就是一个 UUID 字符串;其合法性完全取决于 Redis 里是否存在对应 key。服务端 `DEL` 即撤销。

## 权限校验(仅管理端)

```
1. 取当前管理员 role(来自 JWT subject)
2. 超级管理员(role == 104)直接放行
3. SMEMBERS AUTH_ROLE:{roleId}:apis
4. 对每个元素 "METHOD-path:segment" 还原成 "/path/segment",用 AntPathMatcher 匹配当前请求
5. 命中 → 放行;全部不命中 → 403
```

## 错误响应

### 401 未授权

```json
{ "code": 401, "message": "认证失败" }
```

触发场景:缺少/非法 Authorization 头、AT 签名错误或已过期、`tokenType != access`、管理端 uuid 在 Redis 不存在。

### 403 权限不足

```json
{ "code": 403, "message": "无访问权限" }
```

仅管理端,RBAC 未命中。
