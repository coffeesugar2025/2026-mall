# 快速启动指南

这份文档面向**第一次**在本地运行 ClodRail 的读者。目标:在半小时内让核心链路(注册 / 登录 / 下单)跑起来。

> 想了解具体版本约束,先看 [环境要求](环境要求.md)。

---

## 1. 环境准备

必备中间件(建议用 Docker 一键起):

- MySQL 8.0+
- Redis 7.0+
- RabbitMQ 3.12+
- Nacos 2.2+(**单机模式** + MySQL 持久化)

可选组件(不装会降级或关闭对应功能):

- **Seata Server 2.3.x**(Apache 版,**不是** 1.x):订单 TCC 不可用时回滚会失败
- Elasticsearch 8.x:商城搜索降级为 MySQL LIKE
- XXL-Job Admin 2.4+:订单超时关单、热点预热失效

---

## 2. 初始化数据库

### 2.1 Nacos 配置库

```bash
mysql -u root -p -e "CREATE DATABASE nacos DEFAULT CHARACTER SET utf8mb4;"
mysql -u root -p nacos < Docs/99-部署运维/nacos.sql
```

`nacos.sql` 会写入以下共享配置到 Nacos 的 `config_info` 表:

- `shared-mysql.yaml`、`shared-redis.yaml`、`shared-rabbitmq.yaml`
- `shared-seata.yaml`、`shared-alipay.yaml`

> 💡 **`shared-knife4j.yaml` 和 `shared-langchain4j.yaml` 未包含在脚本里**,需启动 Nacos 后在控制台手动新建(见 4.2)。

### 2.2 业务库 + Seata 库

```bash
mysql -u root -p <<'SQL'
CREATE DATABASE rs_user    DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE rs_ticket  DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE rs_order   DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE rs_mall    DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE rs_assistant DEFAULT CHARACTER SET utf8mb4;
CREATE DATABASE seata      DEFAULT CHARACTER SET utf8mb4;
SQL
```

### 2.3 导入各业务表结构

```bash
cd RailwaySystem-Backend/rs-service
mysql -u root -p rs_user      < rs-user/src/main/resources/sql/user-init.sql
mysql -u root -p rs_ticket    < rs-ticket/src/main/resources/sql/ticket-init.sql
mysql -u root -p rs_order     < rs-order/src/main/resources/sql/order-init.sql
mysql -u root -p rs_mall      < rs-mall/src/main/resources/sql/mall-init.sql
mysql -u root -p rs_mall      < rs-mall/src/main/resources/sql/item.sql          # 商品种子数据
mysql -u root -p rs_assistant < rs-assistant/src/main/resources/sql/assistant-init.sql
```

### 2.4 Seata 服务端表

Seata 2.3.x 以 `db` 模式存事务日志时,需要从 [Seata 官方仓库](https://github.com/apache/incubator-seata/tree/develop/script/server/db)下载 `mysql.sql` 导入 `seata` 库,表包含 `global_table`、`branch_table`、`lock_table` 等。**仅用 `file` 模式可跳过**(修改下文 4.1 的 `shared-seata.yaml` 即可)。

## 3. 启动基础设施

```bash
# Nacos 单机模式(需已改 conf/application.properties 指向 MySQL)
cd nacos/bin && ./startup.sh -m standalone

# Seata Server 2.3.x(可选,订单 TCC 用)
cd seata/bin && ./seata-server.sh
```

MySQL / Redis / RabbitMQ 按你习惯的方式启动即可。

---

## 4. 配置 Nacos

### 4.1 创建命名空间

项目各服务 `bootstrap.yaml` 中硬编码了 namespace ID **`83a90920-a536-4767-b536-3e99c506b1a0`**。你有两种选择:

- **A(推荐初次上手)**:在 Nacos 控制台 → 命名空间 → 新建,把"命名空间 ID"一栏**强制填**上面这个 UUID(字段允许自定义)
- **B**:创建任意 namespace,然后**全局替换**各服务 bootstrap 中的 ID 为你自己的

完成后,Nacos 里会出现一个空的命名空间,shared 配置应显示在这里(2.1 导入的内容)。

### 4.2 补齐 Nacos 里缺失的共享配置

在刚才创建的 namespace 下,**新建以下 2 个配置**(Data ID / Group 默认 `DEFAULT_GROUP`):

- `shared-knife4j.yaml` — API 文档聚合配置,最小内容:
  ```yaml
  knife4j:
    enable: true
    production: false
    setting:
      language: zh_cn
  ```
- `shared-langchain4j.yaml` — AI 客服配置(只有需要启用 AI 客服时必填):
  ```yaml
  langchain4j:
    community:
      dashscope:
        chat-model:
          api-key: ${DASHSCOPE_API_KEY:}
          model-name: qwen-plus
  ```

### 4.3 改写 shared-mysql / shared-redis 等

进入 `shared-mysql.yaml` 等配置,把数据库地址、用户名、密码改成你本地的值。

## 5. 准备各服务私有配置(`bootstrap-dev.yaml`)

每个服务的 `src/main/resources/` 下都提供了 `bootstrap-dev.example.yaml` 示例,**真实使用的 `bootstrap-dev.yaml` 被 `.gitignore` 忽略**,第一次启动必须自己拷贝一份并替换占位符:

```bash
# 以 user-service 为例,其它服务同理
cd RailwaySystem-Backend/rs-service/rs-user/src/main/resources
cp bootstrap-dev.example.yaml bootstrap-dev.yaml
# 编辑 bootstrap-dev.yaml,替换 <MYSQL_PASSWORD>、<REDIS_PASSWORD>、<JWT_KEY> 等占位符
```

涉及的服务:`rs-gateway`、`rs-user`、`rs-ticket`、`rs-order`、`rs-mall`、`rs-assistant`。

## 6. 设置必需的环境变量

用户服务签发 JWT 依赖 **`RS_AUTH_JWT_KEY`**(至少 32 字节的随机字符串),**未设置会启动失败**:

```bash
export RS_AUTH_JWT_KEY="please-replace-with-your-own-random-key-at-least-32-chars"
# 如果生产启 HTTPS,打开 Secure Cookie
export RS_AUTH_REFRESH_COOKIE_SECURE=true
```

可选:

- `DASHSCOPE_API_KEY` — AI 客服的阿里百炼 API Key([免费申请](https://bailian.console.aliyun.com/))
- 支付宝沙箱相关键 — 见 [FAQ Q7](../FAQ.md)

## 7. 编译后端

```bash
cd RailwaySystem-Backend
mvn clean install -DskipTests
```

必须在**根模块**执行一次,以便 `rs-api`、`rs-util/*` 安装到本地 Maven 仓库供其它子模块依赖。

---

## 8. 按顺序启动后端

推荐顺序(前一个服务注册成功再起下一个):

```bash
# 1. 网关
cd RailwaySystem-Backend/rs-gateway           && mvn spring-boot:run
# 2. 用户服务
cd RailwaySystem-Backend/rs-service/rs-user   && mvn spring-boot:run
# 3. 车票服务
cd RailwaySystem-Backend/rs-service/rs-ticket && mvn spring-boot:run
# 4. 订单服务
cd RailwaySystem-Backend/rs-service/rs-order  && mvn spring-boot:run
# 5. 商城服务(非必需)
cd RailwaySystem-Backend/rs-service/rs-mall   && mvn spring-boot:run
# 6. AI 客服服务(非必需)
cd RailwaySystem-Backend/rs-service/rs-assistant && mvn spring-boot:run
```

启动后端口占用确认:

| 组件 | 端口 |
|------|------|
| rs-gateway | 18080 |
| rs-user | 18081 |
| rs-ticket | 18082 |
| rs-order | 18083 |
| rs-assistant(HTTP) | 18084 |
| rs-assistant(Netty WebSocket) | **18085**(硬编码) |
| rs-mall | 18086 |

## 9. 启动前端

```bash
# 用户端
cd RailwaySystem-Frontend/rs-user-web && pnpm install && pnpm dev
# 管理端
cd RailwaySystem-Frontend/rs-admin-web && pnpm install && pnpm dev
```

> 前端默认用 pnpm。如果用 npm 也能跑,但 lockfile 是 pnpm 的。

## 10. 访问验证

| 入口 | 地址 |
|------|------|
| Nacos 控制台 | <http://localhost:8848/nacos>(默认账号 `nacos/nacos`) |
| 网关聚合 API 文档(Knife4j) | <http://localhost:18080/doc.html> |
| 用户端 | <http://localhost:5173> |
| 管理端 | <http://localhost:5174> |

### 健康自检

- Nacos → 服务管理 → 服务列表:能看到 `rs-gateway / user-service / ticket-service / order-service` 等,实例数 > 0、状态健康
- 网关 18080 能访问,且 `/doc.html` 聚合了下游服务的接口
- 注册 + 登录可用(需要 `RS_AUTH_JWT_KEY` 已设置)

## 11. 出错了?

见 [FAQ](../FAQ.md) 的"启动类"和"配置类"小节。最常见三类问题:

1. **Nacos 里找不到共享配置** → namespace 不对,或 `nacos.sql` 未导入目标 namespace
2. **user-service 启动即失败** → `RS_AUTH_JWT_KEY` 未设置,或长度不足
3. **下单触发 TCC 但找不到 Seata TC** → Seata Server 未启动,或 `shared-seata.yaml` 的 `service.default.grouplist` 填错
