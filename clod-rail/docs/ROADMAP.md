# 路线图 ROADMAP

ClodRail 是一个**持续演进**的学习型项目。这份路线图列出了正在做和计划做的事情,让你知道哪些是"现成可用的"、哪些是"规划中的"。

> 想参与任何一项?欢迎提 PR!见 [贡献指南](../CONTRIBUTING.md)。

---

## 🟢 当前版本(v1.0.0)已提供

- [x] 用户服务:注册 / 登录(Access + Refresh 双 Token)/ RBAC / 乘车人
- [x] 车票服务:车站 / 线路 / 车次 / 余票 / 座位库存(**仅支持"线路整段"售票,区间票见下方规划**)
- [x] 订单服务:普通下单 / 热门秒杀(TCC) / 支付宝沙箱 / 退票 / 超时关单
- [x] 积分商城:商品 / 积分兑换 / ES 搜索
- [x] AI 客服:LangChain4j 问答 / Netty WebSocket / 会话记忆 / 人工转接
- [x] 统一网关:Access Token 无状态校验 / 管理端 uuid+Redis Session / RBAC / 路由 / 白名单
- [x] 完整文档体系:架构总览 + 技术选型 + 6 个模块文档 + 3 个亮点专题

## 🚧 近期计划(3 个月内)

### 部署 & DevOps

- [ ] **Docker Compose 一键启动**——`docker-compose up` 拉起全部中间件 + 业务服务
- [ ] **K8s manifests**——提供 Kubernetes 部署示例
- [ ] **CI 流水线**——GitHub Actions 做 Maven 编译 + 前端 build 检查

### 性能 & 观测

- [ ] **JMeter 压测报告**——秒杀场景从 500 → 5000 TPS 的优化记录
- [ ] **SkyWalking 接入**——跨服务链路追踪
- [ ] **Prometheus + Grafana**——基础监控看板

### 功能增强

- [ ] **Sentinel 限流熔断**——网关层接入,替换当前的简单限流
- [ ] **VIP 会员体系**——等级、权益、车次折扣、积分倍率
- [ ] **改签功能**——当前只有退票,改签更复杂(差价、座位重新分配)
- [ ] **区间票(中途站上下车)**——现状只卖"线路起点→终点"整段票,需要将座位库存从"日期+车次"升级为"日期+车次+**区段**",查询按 OD 聚合每一跳最小可售量,下单按选中区段置位。涉及库存模型与订单模型联动改造,优先级中高

---

## 🔮 中期规划(6-12 个月)

### 架构演进

- [ ] **Kafka 替换 RabbitMQ(可选方案)**——大吞吐场景演示
- [ ] **ShardingSphere 分库分表**——订单表按用户 ID 分片
- [ ] **多租户支持**——支持多家铁路公司部署

### AI 能力增强

- [ ] **RAG 知识库**——铁路规章制度作为外部知识源
- [ ] **Agent 工具编排**——改签这种多步任务自动拆解
- [ ] **模型动态路由**——根据问题复杂度选 qwen-turbo/qwen-max
- [ ] **语音客服**——接入 WebRTC + 语音识别/合成

### 前端

- [ ] **移动端 H5 用户端**——基于已有原型图产出正式版
- [ ] **管理端 E2E 测试**——Playwright 覆盖关键流程
- [ ] **暗黑模式支持**——管理端已预留 i18n,补暗色主题

## 💭 长期愿景
