# 开源商城项目汇总



## 电商招聘岗位

```
工作内容：
1. 核心架构设计与开发 (Hands-on Development)
-核心代码撰写：负责平台核心模块（如：复杂询价引擎、订单中心、支付结算、权限体系、ERP接口适配）的架构设计及核心代码编写。
-技术选型与搭建：主导技术栈选型（推荐微服务架构），搭建高可用、高并发、可扩展的基础开发框架，制定代码规范与安全标准。
-难点：亚洲各国多语言、多币种、不同税制、不同付款方式和货运服务提供商。
2. 外包协同与管理 (Outsourcing Management)
-联合设计：与外包供应商共同进行系统详细设计，审核其技术方案、数据库设计及API接口定义，确保符合公司整体架构标准。
-过程管控：管理外包开发进度与质量，通过严格的代码审查 (Code Review)、单元测试覆盖率检查及安全扫描，杜绝外包代码的“黑盒”交付风险。
-知识转移：主导外包团队的知识转移工作，确保所有源代码、文档、部署脚本完整移交，并指导内部团队完全掌握系统逻辑。
3. 需求分析与产品设计 (Product & Requirement)
-业务转化：深入制造业务一线，将复杂的线下采购/销售流程转化为清晰的技术需求文档 (PRD) 和技术规格书。
-英语沟通：作为技术接口人，直接用英语与海外客户或跨国供应商沟通需求，编写英文技术文档及API文档。
4. 自主运维与持续迭代 (Autonomous Maintenance & Ops)
-运维体系搭建：平台上线后，负责建立自主运维体系，包括CI/CD流水线、自动化测试、日志监控 (ELK/Prometheus)、报警机制及灾备方案。
-故障响应与修复：作为最终技术责任人 (On-call)，直接处理生产环境的紧急故障、性能瓶颈及安全漏洞，不再依赖外包。
-持续迭代：根据业务反馈，自主规划并执行平台的二期、三期功能迭代，逐步降低对外部资源的依赖，实现完全自主可控。

工作要求：
1.教育背景：
-计算机科学、软件工程或相关专业本科及以上学历。
2.行业与经验（硬性指标）：
-8年以上开发经验，其中至少3年担任技术负责人或架构师角色。
-必须具备制造业/供应链行业背景，熟悉To B电商复杂交易逻辑（询比价、合同、账期、多级审批等）。
-必须有主导过To B平台从0到1全流程的经验，且包含管理外包团队的成功案例。
3.技术硬实力：
-精通后端开发：熟练掌握 Java (Spring Cloud/Boot) 或 Go/Python 等主流语言，有大规模微服务架构设计经验。
-全栈能力：熟悉前端技术 (Vue/React/Angular)，能独立进行前端架构评审或直接开发核心页面。
-数据库与中间件：精通 MySQL/PostgreSQL，熟悉 Redis, Kafka/RabbitMQ, Elasticsearch 等在电商场景的应用。
-DevOps能力：精通 Docker, Kubernetes (K8s), CI/CD (Jenkins/GitLab CI)，具备搭建和维护自动化运维平台的能力。
4.外包管理经验：
-具备极强的代码鉴赏力和技术把控力，能通过Code Review有效识别外包代码的质量隐患、安全漏洞及逻辑缺陷。
-善于管理外部供应商的交付预期，确保技术资产（代码、文档）的完整性和可维护性。
5.语言能力：
-英语流利（听说读写精通），能作为工作语言进行技术讨论、文档编写及跨国会议。
6.综合素质：
-具备“主人翁”意识，愿意对系统的长期稳定性和可维护性负责，不仅仅是“交付即走”。
-优秀的沟通协调能力，能在业务、内部团队、外包团队三方之间高效流转信息。
```
